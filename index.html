⚠️ قبل از شروع بخوان

🖥️
سرور VPS خارج از ایران
برای Full Tunnel به یه VPS نیاز داری — هلند، آلمان یا ترکیه. بدون VPS فقط HTTP relay داری و تلگرام و بقیه اپ‌های TCP کار نمی‌کنن.
🔒
فیلترشکن برای راه‌اندازی اولیه
فقط برای deploy کردن Apps Script در script.google.com به فیلترشکن نیاز داری. بعد از راه‌اندازی کامل دیگه نیازی نیست.
✅
چرا Full Tunnel؟
ترافیک همه اپ‌ها — تلگرام، مرورگر، بانک، یوتیوب — از VPS شخصی تو رد میشه. ISP فقط اتصال به www.google.com می‌بینه و نمی‌تونه بلاک کنه.

  • 📱 اندروید با نسخه 7 به بالا

⚙️ تنظیمات بهینه Full Tunnel (مهم!)

در اپ mhrv-rs این تنظیمات رو اعمال کن:

  • 1️⃣
    Mode: Full tunnel (no cert)همه ترافیک — بدون certificate — از VPS رد می‌شه.
  • 2️⃣
    Block QUIC: ✅ روشنUDP/443 بلاک می‌شه — مرورگرها به TCP/HTTPS می‌افتن. بدون این تنظیم tunnel crash می‌کنه.
  • 3️⃣
    Block DoH: ✅ روشنDNS over HTTPS بلاک می‌شه — DNS از تونل رد می‌شه. جلوی DNS leak رو می‌گیره.
  • 4️⃣
    SNI Tester (مهم‌ترین!)هر SNI رو تست کن. فقط ۲-۳ تا با کمترین ping انتخاب کن. بقیه رو بردار. سرعت ۲ برابر می‌شه.
  • 5️⃣
    Google IP: 216.239.38.120یا Auto-detect بزن. اگه SNI همه fail شد این IP رو تغییر بده: 216.239.32.0 یا 216.239.36.0
  • 6️⃣
    Front Domain: www.google.comدست نزن — ISP این رو می‌بینه و فکر می‌کنه به گوگل وصل شدی.
  • 7️⃣
    parallel_relay: 1 (پیش‌فرض)اگه اتصال ناپایداره، 2 امتحان کن. بیشتر از 3 نزن — quota سریع تموم می‌شه.
  • 8️⃣
    چند Deployment IDهر جیمیل = یه Deployment = سهمیه جداگانه. ۳ تا جیمیل = سرعت و quota ۳ برابر.

🔧 عیب‌یابی Apps Script

  • 🔴
    504 Relay timeout / Apps Script unresponsivequota روزانه تموم شده (۲۰k call/روز). راه‌حل: جیمیل دیگه + deploy جدید. یا فردا صبر کن.
  • 🔴
    Syntax Error deployخط آخر اضافه رو پاک کن. یا کد رو دوباره paste کن.
  • 🔴
    tunnel bad base64Apps Script خراب شده — دوباره deploy کن.
  • 🔴
    invalid front_domainفیلد front domain باید www.google.com باشه.
  • 🟡
    unauthorized در لاگAUTH_KEY در VPS و Apps Script باید دقیقاً یکی باشن.
  • 🟡
    batch failed: timeout (زیاد)SNI مناسب نیست. SNI Tester بزن، فقط ۲-۳ تا با کمترین ping نگه دار.
  • 🟡
    sni_probe همه FAILGoogle IP رو عوض کن: 216.239.32.0 یا Auto-detect بزن.
  • 🟡
    اتصال قطع و وصلdeployment معیوب خودکار ۲ دقیقه blacklist می‌شه — طبیعیه. چند deployment داشته باش.
  • 🟢
    App not installed هنگام آپدیتSettings → Apps → mhrv-rs → Uninstall → نصب مجدد (یه بار)
  • 🟢
    VPS قفل شدپورت SSH رو قبل از ufw enable باز کن. قفل شد: OS reinstall لازمه.
  • 🟢
    Docker error (dpkg)dpkg --configure -a بزن، بعد: apt install -y docker.io

📊 توضیح لاگ‌های اپ mhrv-rs

از منوی اپ → Logs رو باز کن. هر پیام یه معنی داره:

  • 🟢
    tunnel session opened✅ اتصال موفق برقرار شد.
  • 🟢
    dispatch X → blocked (block_doh)عمدیه! اپ DNS over HTTPS رو بلاک می‌کنه تا DNS لو نره. نشانه سلامت اپ.
  • 🟢
    dispatch X → full tunnel (via batch mux)✅ ترافیک از تونل رد شد.
  • 🟢
    h2 connection established✅ اتصال HTTP/2 به Apps Script برقراره.
  • 🟢
    sni_probe: X ok in Xmsاین SNI خوبه، نگهش دار. کمتر از ۲۰۰ms عالیه.
  • 🟡
    WARN ...mux...چند اتصال در یک batch — سرعت بهتر. کاملاً طبیعی.
  • 🟡
    batch failed: timeoutیه batch بعد از ۳۰ ثانیه جواب نگرفت. اپ retry می‌کنه. اگه زیاد ببینی → SNI بهتر انتخاب کن.
  • 🟡
    sni_probe: X FAILاین SNI از ایران بسته‌ست. تیکش رو بردار. اگه همه FAIL → IP گوگل رو عوض کن.
  • 🔴
    tunnel connect_data error: timeoutquota تموم / AUTH_KEY اشتباه / سرور قطع. چک کن کدوم مشکله.
  • 🔴
    Relay timeout — Apps Script unresponsivequota پر شده. یه Deployment جدید روی جیمیل دیگه بساز.
